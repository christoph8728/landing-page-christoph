# Personal Landing Page - Project Documentation

## Architecture Philosophy

This is a **content-driven, configuration-first** personal website built with Next.js. The core principle is: **everything editable lives in YAML config or markdown files** — never in React components.

## Single Source of Truth

**`site.config.yaml`** is the central configuration file. It controls:

- **Identity & Author Info**: Name, tagline, headline, email, location, socials
- **Content**: Thesis, numbers/stats, speaking topics, CTA text
- **Section Toggles**: Show/hide entire sections (thesis, track_record, numbers, etc.)
- **Features**: Blog, RSS, scheduled posts, drafts, reading time, analytics
- **Design**: Colors, fonts, max-width
- **Analytics**: Plausible or Umami configuration
- **Navigation**: Menu items with conditional rendering

### Content Sections in YAML

```yaml
content:
  thesis: |
    The thesis paragraph with HTML support (<strong>tags allowed</strong>)

  numbers:
    - value: "10+"
      label: "Years in operations"

  speaking_topics:
    - title: "Topic Title"
      format: "Keynote · 25 min"
      description: "Description text"

  cta:
    heading: "Let's talk."
    description: "CTA description"
```

## Content Files Structure

All editable content lives in `content/` directory as markdown files:

### Track Record (`content/track-record/`)
Career history, sorted by date (newest first):
```yaml
---
title: "COO & Co-Founder"
organization: "CloudStage GmbH · Munich"
period: "2025 – now"
date: 2025-01-01  # for sorting
description: "Role description"
tags:
  - Tag1
  - Tag2
---
```

### Publications (`content/publications/`)
Academic and professional publications:
```yaml
---
title: "Publication Title"
authors: ["Author 1", "Author 2"]
year: 2019
venue: "Conference/Journal Name"
type: "Journal Article" | "Conference Paper" | "Doctoral Thesis"
doi: "10.xxxx/xxxxx"
url: "https://..."
highlight: true  # Shows in highlighted section
date: 2019-01-01  # for sorting
---
```

### Speaking Engagements (`content/speaking/`)
Past and upcoming speaking events:
```yaml
---
title: "Talk Title"
event: "Event Name"
location: "City, Country"
type: "Keynote" | "Panel" | "Workshop"
date: 2026-06-15
---
```

### Blog Posts (`content/blog/`)
Blog articles with frontmatter:
```yaml
---
title: "Post Title"
date: 2026-04-10
tags: [tag1, tag2]
draft: false
summary: "Optional custom summary for SEO, social sharing, and previews"
og_image: "/images/custom-og.png"  # Optional custom OG image
---

Post content in markdown...
```

**Important**:
- Posts with `draft: true` are hidden when `features.drafts` is enabled
- Posts with future dates are hidden when `features.scheduled_posts` is enabled
- Reading time is auto-calculated when `features.reading_time` is enabled
- `summary` field is optional:
  - If provided, it's used for previews, RSS feeds, and SEO
  - If empty, a summary is auto-generated by truncating the body to ~160 chars
  - Auto-generated summaries strip markdown formatting and truncate at word boundaries
- `og_image` field is optional:
  - If provided, uses your custom image for social sharing (Open Graph, Twitter Cards)
  - If empty, auto-generates a branded card using the site's design and post title
  - Generated cards use `/api/og` route with Next.js OG image generation
  - Cards are 1200x630px (standard social media size)

## Key Design Decisions

### 1. Content-Driven Architecture
React components should NEVER contain hardcoded content. All content comes from:
- `site.config.yaml` for static content
- `content/*.md` files for structured data

### 2. Dynamic Rendering
- Blog mode: `dynamic` (default) | `static` | `isr` (set in config)
- Sections auto-hide when empty (e.g., engagements, publications)

### 3. HTML Support in Config
Some config fields support HTML:
- `site.headline`: Use `<span class="hl">...</span>` for orange highlight, `<br />` for line breaks
- `content.thesis`: Use `<strong>` tags for bold text

### 4. Conditional Navigation
Nav items can have a `requires` field that references a config path:
```yaml
nav:
  - label: "Publications"
    href: "/#publications"
    requires: "sections.publications"  # Only shows if sections.publications is true
```

## Common Editing Tasks

### Change Hero Section
Edit in `site.config.yaml`:
```yaml
site:
  tagline: "Your tagline"
  headline: "Your headline<br />with <span class=\"hl\">highlight</span>"
  description: "Your description"
```

### Add/Edit Track Record
Create/edit files in `content/track-record/`:
- Filename doesn't matter (slug is auto-generated)
- Use `date` field for sorting (newest first)
- `period` is for display (e.g., "2025 – now")

### Add Blog Post
Create new file in `content/blog/`:
- Filename format: `YYYY-MM-title.md`
- Set `draft: false` to publish
- Date controls visibility if `scheduled_posts` is enabled

### Change Colors/Design
Edit `design` section in `site.config.yaml`:
```yaml
design:
  font_body: "Outfit"
  color_accent: "#9F580A"     # Orange highlight color
  color_bg: "#F7F6F3"
  max_width: "820px"
```

### Toggle Sections
Enable/disable entire page sections:
```yaml
sections:
  thesis: true
  track_record: true
  numbers: true
  speaking_topics: true
  publications: true
  blog_preview: true
```


## Important Notes

- **Never edit `src/app/page.tsx`** for content changes — use `site.config.yaml` or content files
- All dates should be in `YYYY-MM-DD` format
- Arrays in YAML use `- item` syntax
- Use double quotes for strings with special characters in YAML
- Reading time is auto-calculated at ~220 words/minute
- RSS feed auto-generates from blog posts
- Blog post summaries:
  - Available via `post.summary` in TypeScript
  - Can be used for previews, RSS descriptions, Open Graph tags, etc.
  - Auto-generated if not provided (strips markdown, truncates to ~160 chars)

## Analytics Setup

Configured in `site.config.yaml`:
```yaml
analytics:
  provider: "plausible"  # or "umami" or null
  plausible:
    domain: "yourdomain.com"
    script_url: "https://plausible.io/js/script.js"
```

## Content Filtering Logic

See `src/lib/content.ts` for filtering rules:
- Draft posts hidden when `features.drafts = true`
- Future-dated posts hidden when `features.scheduled_posts = true`
- Posts sorted by date (newest first)
- Publications can be `highlight: true` for featured section

## Open Graph Image Generation

The site auto-generates Open Graph images for social sharing using `@vercel/og`:

### How It Works

1. **API Route**: `/api/og` (src/app/api/og/route.tsx)
   - Accepts query params: `title` and `type` (blog/home)
   - Uses site design colors from `site.config.yaml`
   - Generates 1200x630px images on-the-fly

2. **Automatic Generation**:
   - Homepage: Uses `site.headline` as title
   - Blog posts: Uses post title (unless custom `og_image` provided)

3. **Custom Images**:
   - Add `og_image: "/path/to/image.png"` to blog post frontmatter
   - Image will be used instead of auto-generated card

4. **Design**:
   - Background: `design.color_bg`
   - Text: `design.color_text`
   - Muted text: `design.color_text_muted`
   - Border: `design.color_border`
   - Includes site branding and location

- Blog post: `http://localhost:3000/api/og?title=Post%20Title&type=blog`
